# 初賽提案大綱

## 題目

運用 AI 打造 iRent 智能車況管家：優化租還車體驗與營運效率

## 一句話方案

把租客還車照片、語音備註、車聯網訊號與歷史工單交給多模態 AI 判讀，產生車況風險分數、處置建議、客服摘要與派工優先序，讓車輛能更快回到可租狀態。

## 痛點

- 共享車租還車高周轉，人工逐台檢查容易延遲下一筆訂單。
- 車損、髒污、異味與電量不足常在租客端、客服端、營運端之間重複確認。
- 高風險車輛沒有被即時下架或改派，會放大客服爭議與營運成本。

## 目標使用者

- 租客：快速完成還車，降低被誤判責任的焦慮。
- 營運人員：依風險分數安排清潔、充電、維修與複拍。
- 客服：用 AI 摘要與佐證資料回覆爭議案件。
- 調度系統：在下一筆訂單前決定保留、改派或下架。

## AI 工作流

1. 租客上傳四角照片、內裝照片與語音備註。
2. Vision model 比對前後車況，偵測刮痕、凹痕、垃圾、儀表燈號與清潔狀態。
3. LLM 整合車聯網訊號、歷史維修、租借紀錄與下一筆訂單時間。
4. 產出風險分數、原因摘要、建議動作與客服溝通草稿。
5. 依 SLA 自動建立派工任務或改派鄰近車輛。

## Prototype 交付

- Web dashboard：車況風險排序、AI 摘要、KPI 與處置建議。
- Demo 情境：三台車分別代表可上架、需複檢、需下架維修。
- 下一版可接：真實圖片上傳、OpenAI Vision API、簡易派工規則引擎、客服訊息產生。

## 預期成效

- 降低人工驗車時間。
- 提升下一筆訂單保留率。
- 減少客服爭議處理時間。
- 縮短車輛非計畫下架時間。

## 初賽評分對應

- 主題連結度：直接回應 iRent 租還車體驗與營運效率。
- 創意表現：把「車況檢查」升級成跨租客、客服、營運、調度的 AI 決策流程。
- 提案可行性：先以照片與表單資料啟動，逐步接入車聯網與工單資料。
- AI 應用整合性：多模態辨識、LLM 摘要、風險分數與自動化派工共同運作。
